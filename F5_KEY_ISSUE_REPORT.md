# 🔍 F5录音键失效问题诊断报告

## 问题描述
进入本地LLM项目并启动服务后，Mac的F5录音键（语音转文字功能）失效。

## 可能的原因分析

### 1. **端口冲突**
- 项目可能占用了系统语音服务使用的端口
- 特别是5000端口（AirPlay/音频流服务常用端口）
- 控制面板使用5005端口

### 2. **音频设备占用**
- Python进程可能锁定了音频输入设备
- Docker容器可能影响音频服务
- 后台服务可能干扰了系统音频

### 3. **系统权限问题**
- 麦克风权限被占用或阻塞
- 辅助功能权限冲突
- 听写服务被中断

### 4. **进程冲突**
- FastAPI服务（端口8000）
- Ollama服务（端口11434）
- Weaviate服务（端口8080）
- 控制面板服务（端口5005）

## 诊断步骤

### 自动诊断
```bash
# 运行诊断脚本
python diagnose_f5_issue.py
```

### 手动检查
1. **检查音频服务状态**
   ```bash
   ps aux | grep coreaudiod
   ```

2. **检查端口占用**
   ```bash
   lsof -i :5000
   lsof -i :5005
   ```

3. **检查Python进程**
   ```bash
   ps aux | grep python | grep -E "(audio|speech|voice)"
   ```

## 解决方案

### 快速修复
```bash
# 运行修复脚本
./fix_f5_key.sh
```

### 手动修复步骤

1. **停止项目服务**
   ```bash
   ./control_center.sh stop
   ```

2. **重启音频服务**
   ```bash
   sudo killall coreaudiod
   ```

3. **清理可能的冲突**
   ```bash
   # 停止所有Docker容器
   docker compose down
   
   # 终止相关Python进程
   pkill -f "python.*audio"
   pkill -f "python.*speech"
   ```

4. **重置语音识别**
   - 系统偏好设置 → 键盘 → 听写
   - 关闭听写功能
   - 等待5秒
   - 重新开启听写功能

5. **检查快捷键设置**
   - 系统偏好设置 → 键盘 → 快捷键
   - 确保F5没有被其他功能占用

## 预防措施

### 1. 修改项目配置
避免使用可能冲突的端口：
- 避免使用5000端口
- 使用非标准端口范围（如8000-9000）

### 2. 优化启动脚本
在`control_center.sh`中添加检查：
```bash
# 检查音频服务状态
check_audio_service() {
    if ! pgrep coreaudiod > /dev/null; then
        echo "警告: 音频服务未运行"
        sudo killall coreaudiod 2>/dev/null
        sleep 2
    fi
}
```

### 3. 使用隔离环境
- 优先使用Docker容器运行服务
- 确保容器不映射音频相关端口
- 使用--no-audio标志（如果有）

2. **修改服务配置**
